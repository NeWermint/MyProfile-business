Приложение "MyProfile для предпринимателей" 

Описание проекта


Данный проект представляет собой бизнес-версию консольного приложения "MyProfile". Его основная цель — предоставить предпринимателям удобный инструмент для хранения и вывода ключевой личной, регистрационной и банковской информации, что упрощает заключение договоров и дополнительных соглашений.


Основные изменения и задачи


В процессе доработки приложения были выполнены следующие ключевые задачи:


Удаление устаревшего функционала: Удален ввод, хранение и вывод информации о социальных сетях и мессенджерах.


Добавление регистрационных данных: Добавлен ввод полей ОГРНИП и ИНН.



Проверка ОГРНИП: Поле ОГРНИП должно содержать строго 15 цифр. В случае некорректного ввода, пользователю предлагается ввести значение заново.





Добавление банковских реквизитов: Добавлен ввод расчётного счёта, названия банка, БИК и корреспондентского счёта.


Проверка Расчётного счёта: Поле должно содержать строго 20 цифр. В случае некорректного ввода, пользователю предлагается ввести значение заново.




Добавление почтового адреса: Добавлен ввод и вывод почтового адреса и индекса.


Фильтрация Индекса: Индекс вводится как произвольная строка, но приложение сохраняет только цифры.




Обновление вывода информации: Обновлен вывод полной информации, включающий все новые поля (ОГРНИП, ИНН, банковские реквизиты).




Структура меню приложения



Меню приложения адаптировано под бизнес-версию:


1. Ввести или обновить информацию 


1 - Личная информация (Имя, Возраст, Телефон, E-mail, Индекс, Почтовый адрес, Доп. информация).




2 - Информация о предпринимателе (ОГРНИП, ИНН, Расчётный счёт, Название банка, БИК, Корреспондентский счёт).



0 - Назад.

2. Вывести информацию 


1 - Личная информация (вывод основных полей) .



2 - Вся информация (вывод всех личных, регистрационных и банковских данных).



0 - Назад.

0. Завершить работу 






Технические требования

Язык и среда: Приложение реализовано на Python 3.x.


Зависимости: Не используются сторонние библиотеки.


Стандарты кодирования (PEP 8): Код соответствует стилю написания PEP 8.


Переменные должны быть развернутыми и описательными (например, numbers вместо num).

Обработка ввода:

Поля: Возраст, ОГРНИП, ИНН, Расчётный счёт, БИК, Корреспондентский счёт вводятся как целые числа.

Корректно проверяются условия для: возраста (неотрицательное число), ОГРНИП (15 цифр) и Расчётного счёта (20 цифр).
